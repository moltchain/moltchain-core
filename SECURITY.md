# Security Policy

## Reporting a Vulnerability

The Moltchain team takes security seriously. We appreciate responsible disclosure of security issues.

### How to Report

**DO NOT** open public issues for security vulnerabilities.

Instead, please report via:

1. **Email**: security@moltchain.xyz
2. **Encrypted**: Use our PGP key (available on request)

### What to Include

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Your contact information

### Response Timeline

- **Initial Response**: Within 24 hours
- **Status Update**: Within 72 hours
- **Resolution**: Depends on severity

## Bug Bounty Program

We offer rewards for responsible disclosure:

| Severity | Reward | Examples |
|----------|--------|----------|
| Critical | $10,000 - $50,000 | Fund loss, contract takeover |
| High | $5,000 - $10,000 | Privilege escalation, DoS |
| Medium | $1,000 - $5,000 | Information disclosure |
| Low | $100 - $1,000 | Minor issues |

### Eligibility

To qualify:
- Be the first to report the issue
- Provide sufficient detail to reproduce
- Don't exploit the vulnerability
- Don't disclose publicly before fix
- Follow responsible disclosure practices

### Scope

**In Scope:**
- Evolution contract
- Agent registry
- Bridge contracts
- SDK vulnerabilities
- Infrastructure issues

**Out of Scope:**
- Third-party dependencies
- Known issues (check existing issues first)
- Social engineering
- Physical attacks

## Security Measures

### Smart Contracts

- Multiple professional audits
- Formal verification for critical functions
- Bug bounty program
- Time-locks on sensitive operations
- Emergency pause mechanisms

### Development

- Secure development practices
- Code review requirements
- Automated security scanning
- Dependency audits

### Operations

- Multi-signature wallets
- Secure key management
- Regular security audits
- Incident response procedures
